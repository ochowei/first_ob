## 🧩 遊戲設計：故事與體驗

### 1. 立項即設計核心
- 決定題材＝決定目標族群
- 題材決定行銷策略與社群互動形式

### 2. 世界觀需與劇情深度綁定
- 避免「可替換的世界觀」
- 好世界觀要能推動劇情與衝突發展

### 3. 角色設計 = 衝突設計
- 每個角色都需有「慾望」與「阻力」
- 衝突包含內在掙扎與角色關係張力

### 4. 主角個性與參與感
- 無個性的主角（如 Coffee Talk）會削弱劇情感染力
- 建議給予主角基本人格設定，推動情節與互動

## 🎯 立項與行銷策略

### 1. 精準定位比全方位重要
- 小團隊應採取「小而準」策略，明確受眾與規模

### 2. 行銷從立項那一刻開始
- 遊戲的賣點、風格與敘事方式應一開始就考量到行銷

### 3. 題材與玩法的黃金組合

> ✅ **大眾題材 + 小眾玩法**  
> 🚫 小眾題材 + 大眾玩法

#### 解釋：
- **大眾題材**吸引玩家點擊（例如亡靈、親情、戀愛）
- **小眾玩法**留下玩家與創造差異化（如敘事式烹飪模擬）
- 相反組合常導致乏人問津或風格不協調

### 4. 參展、競賽與 Demo 策略
- 若遊戲氣氛需靜心體驗，不建議參展
- 參賽僅供曝光，無需依賴結果
- Demo 應選在曝光高峰（如 Steam 新品節）
  - 展示「核心體驗」
  - 保留部分內容，引發期待

### 5. Early Access 的使用情境
- 適合 EA：開放世界、Roguelike 等可反覆遊玩的類型
- 不建議 EA：一次性劇情型遊戲（會劇透、削弱後續銷售）

## 📌 實務表格建議：題材 x 玩法發想

| 題材（吸引人點進來） | 搭配玩法（留下來的理由） |
|------------------------|---------------------------|
| 亡靈吃拉麵             | 模擬經營 + 敘事           |
| AI陪伴型社會           | 多角色視角 + 解謎         |
| 青春戀愛與回憶         | 選擇導向劇情 + 回溯系統   |
| 社區中的神秘事件       | 環境探索 + 敘事對話       |

---

## ✅ 結語三重點

1. 設計前先問自己：「這個故事要對誰說？」
2. 世界觀 ≠ 背景裝飾，而是推進劇情的引擎
3. 行銷不是最後一步，而是從 Day 1 就開始的思考
